# 데이터 출처 및 사용 주의사항

## 범위

모델 학습과 서비스의 기준 데이터는 사용자가 제공한 `batting_stats.csv`,
`pitching_stats.csv` 두 파일이다. 선수 식별에는 원본의 KBO `player_id`를 사용하며,
`source_url`은 원본 선수 페이지를 추적하기 위한 provenance 필드로 보존한다.

2026 시즌 기록은 KBO 공식 선수 기록 페이지에서 2026-07-20에 별도 수집한 진행 중
snapshot이다. `IsPartial=true`와 `AsOfDate`를 함께 저장하며, 완결 시즌 데이터와 혼동하지
않도록 별도 정제 폴더와 import batch로 DB에 적재한다. 조회 API에는 포함하지만 기존
1982~2025 ML 학습 파일에는 합치지 않는다. 수집기는 실행 시 robots.txt를 확인하고 요청
사이에 최소 1초 간격을 둔다.

2026 구단 로스터는 KBO 공식 `선수 등록 현황`에서 2026-07-20 기준으로 수집했다. 이는
구단 소속 선수 전체가 아니라 해당 날짜의 1군 등록 snapshot이며 감독·코치는 제외한다.
로스터도 요청 간격 1초 이상, robots.txt 확인, 원본 CSV 비공개 정책을 동일하게 적용한다.

## 저장소 정책

- `data/raw` 원본 CSV는 `.gitignore`와 `.dockerignore`로 공개 저장소 및 이미지에서 제외한다.
- 2026 snapshot도 원본 통계이므로 Git에서 제외하고, 저장소에는 재현용 수집 코드와
  행 수·누락·중복을 기록한 품질 요약만 보관한다.
- 배포 환경의 2026 가치 랭킹 재현을 위해 `data/processed/2026`의 정규화된 타자·투수
  파일은 보관하며, 수집 기준일이 바뀔 때 전처리 명령으로 함께 갱신한다.
- `data/processed`에는 전처리된 통계가 있으므로 공개 전 원데이터 제공처의 이용약관과
  재배포 허용 범위를 별도로 확인해야 한다.
- 모델 artifact와 예측 결과도 원천 통계에서 파생된 데이터이므로 동일한 확인이 필요하다.
- 이 프로젝트는 KBO 또는 구단의 공식 서비스가 아니며, 통계의 공식성을 보증하지 않는다.

## 공개 전 확인 항목

1. CSV를 수집·제공한 사이트와 수집 일자를 README에 기록한다.
2. 원본 및 가공 통계의 저장·가공·공개·상업적 이용 허용 범위를 확인한다.
3. 허용 범위가 불명확하면 공개 저장소에는 코드와 작은 예제 데이터만 두고 전체 데이터는
   사용자가 직접 준비하도록 안내한다.
4. 선수 평가 결과가 사실 단정이 아닌 통계 모델의 추정치임을 UI와 모델 카드에 표시한다.

현재 저장소만으로는 원본 제공처의 구체적인 라이선스를 확정할 수 없다. 따라서 이 문서는
사용 허가를 부여하지 않으며, 실제 공개·배포 전 별도 확인이 필요하다.
